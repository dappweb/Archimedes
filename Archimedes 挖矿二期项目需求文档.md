# **Archimedes 挖矿二期项目需求文档**

## **1\. 项目概述**

本项目为Archimedes挖矿平台二期，主要围绕 **ARC** (挖矿产出代币) 和 **DES** (生态权益代币) 构建。系统包含USDT入金分润、算力挖矿、动态层级奖励、交易税费分配及独特的出局/复投机制。

## **2\. 代币经济模型 (Tokenomics)**

### **2.1 ARC 代币 (核心产出)**

* **代币名称**: ARC  
* **总发行量**: 1亿枚  
* **分配比例**:  
  * **2%**: 底池 (LP)  
  * **60%**: 挖矿产出  
  * **38%**: 销毁产出（质押销毁）

### **2.2 DES 代币 (生态权益/手续费)**

* **代币名称**: DES  
* **总发行量**: 1亿枚  
* **分配比例**:  
  * **私募**: 221.7万枚 (后台可手动修改额度)  
  * **LP底池**: 25万枚  
  * **销毁**: 剩余部分（1亿 \- 私募 \- LP）全部销毁  
* **释放机制**:  
  * 上线即开通买卖。  
  * 私募部分：自上线日起，每日释放 **1%**，共100天释放完毕。

## **3\. 交易对与税费规则 (Swap Logic)**

### **3.1 ARC/USDT 交易对**

* **买入**:  
  * **费率**: 0%  
  * **权限**: 仅 **白名单地址** 可买（前期限制，后期开放）。  
* **卖出**:  
  * **费率**: 5%  
  * **权限**: 所有用户可卖。  
  * **税费分配**:  
    * **1.5%**: 分配给节点 (至尊节点/V3及以上)  
    * **1.5%**: 分配给小节点 (超级节点/V2)  
    * **2.0%**: 直接销毁

### **3.2 DES/USDT 交易对**

* **买入**: 费率 0%  
* **卖出**: 费率 3% (全部销毁)  
* **依赖**: 手续费计算需实时获取DES价格。

## **4\. 挖矿入金业务逻辑 (USDT Inflow)**

用户投入 USDT 购买矿池算力，资金按以下比例实时分配：

| 比例 | 资金去向/用途 | 详细逻辑说明 |
| :---- | :---- | :---- |
| **5%** | 至尊节点分红 | 每天定时均分给 **V3及以上** 用户。 |
| **65%** | ARC合约池 | **回购销毁机制**：每1小时执行一次，使用该池内 **5%** 的资金，按实时价格回购ARC并销毁（用户主动触发）。 |
| **10%** | 社区办公室奖励 | 见下方 \[4.1 社区办公室逻辑\]。 |
| **10%** | 平台官方钱包 | 归属平台（后期规划：5%进LP，5%销毁）。 |
| **10%** | 购买 DES 销毁 | 按实时价格购买 DES 并销毁。 |

### **4.1 社区办公室逻辑 (10%)**

* **定义**: 平台指定若干地址为“社区办公室”（后台可设置）。  
* **分配规则**:  
  * 向上追溯用户的上级链。  
  * 找到**最近的一个**属于“社区办公室”的地址，给予奖励。  
  * **截断机制**: 只要找到最近的一个，更上级的办公室不再获得奖励（例如 A\>B\>C，A、B均为办公室，C入金，奖励给B，A无奖励）。  
  * 该地址可在前端自行提币。

## **5\. 挖矿产出与结算 (Rewards Output)**

### **5.1 总产出控制**

* **每日总产出**: 合约写死 **30,000 ARC**。  
* **分配结构**: 15,000 (静态) \+ 15,000 (动态)。  
* **结算方式**: 每日产出转至中心化钱包，由平台在每晚 22:00 根据算力和等级进行手动/脚本分配和销毁。

### **5.2 静态奖励 (算力挖矿)**

* **基础算力**: 初始 1 USDT \= 1 算力。  
* **时间加成**: 每晚进场1天，单位USDT获得的算力增加 **3% (0.03)**。  
  * *例*: D1投入100U=100算力; D2投入100U=103算力。  
* **质押销毁加成**: 销毁ARC可获得算力倍增（占用3800万销毁产出池）。  
  * 销毁 100 ARC \-\> 1.1倍算力  
  * 销毁 200 ARC \-\> 1.2倍算力  
  * ...  
  * 销毁 1000+ ARC \-\> 2.0倍算力  
  * *规则*: 销毁量为用户持有ARC总量的1%，剩余LP底池200万手动转入。

### **5.3 动态奖励 (市场推广)**

基于静态收益进行计算，包含三类奖励：

#### **A. 直推奖励 (10层)**

* **门槛**: 必须直推 **10人** 且均已入金。  
* **深度**: 推1人拿1层，推10人拿10层。  
* **奖励**: 拿对应层级用户 **静态收益的 2%**。

#### **B. 小区奖励 (级差 \+ 推广)**

* **小区业绩定义**: 伞下所有用户入金总额 \- (最大一条线的业绩)。  
* **奖励构成**:  
  1. **级差奖励**: 拿伞下用户的静态收益差值 (例如 A是V3 30%, B是V2 20%, A拿B伞下静态收益的 10%)。  
  2. **推广奖励**: 拿伞下等级用户静态收益 \* 奖励系数。  
* **封顶**:  
  * 最高产出为静态奖励的 **20%** (针对个人? 需确认，此处文档表述为“最高产出：静态奖励的产出20%”)。  
  * V8等级 (999亿业绩) 仅限平台地址，系数75%。

#### **C. 同级奖励 (平级)**

* **门槛**: **V3 及以上** 才能触发。  
* **规则**: 当上级与下级等级相同时，上级可拿该下级静态收益的 **5%**。

| 等级 | 小区业绩 (USDT) | 奖励系数 | 备注 |
| :---- | :---- | :---- | :---- |
| **V1** | 1万 | 10% |  |
| **V2** | 3万 | 20% | 小节点/超级节点基准 |
| **V3** | 10万 | 30% | 平级+5%，至尊节点基准 |
| **V4** | 30万 | 40% |  |
| **V5** | 100万 | 50% |  |
| **V6** | 300万 | 60% |  |
| **V7** | 1000万 | 70% |  |
| **V8** | 999亿 | 75% | 仅限平台账号 |

### **5.4 出局与复投机制**

* **3倍出局**: 当 (静态收益 \+ 动态收益) 累计达到投入本金的 **3倍** 时，停止产出。  
* **复投规则**:  
  * 必须复投才能继续提取收益。  
  * 复投比例: **50% USDT \+ 50% ARC** (按市场价计算算力)。  
  * 复投后：静态算力清零重算，动态网体关系保留。  
* **特例**: 平台账号永不出局。

## **6\. 提现与领取规则 (Withdrawal)**

### **6.1 手续费逻辑 (强关联DES)**

用户提取ARC奖励时，需支付DES作为手续费。

* **公式**: 手续费 \= (提取金额价值 \* 5% / DES价格) \+ 3 DES。  
  * *即: 价值5%的DES \+ 固定3个DES* (固定数量后台可调 0.1-3)。

### **6.2 额度判断逻辑**

* **条件 A**: 用户账户内DES余额 \>= 手续费  
  * \-\> **允许提取**，扣除DES。  
* **条件 B**: 用户账户内DES余额 \< 手续费  
  * \-\> **无法提取**，提示“请继续充值DES”。

## **7\. 后台管理系统需求 (Admin Panel)**

1. **仪表盘/数据统计**:  
   * ARC每日产出统计。  
   * 每日收益记录、销毁记录。  
   * DES释放记录（每日每个用户）。  
   * 质押记录（U的质押、当前算力）。  
2. **参数配置**:  
   * **每日静态产出量配置** (需每日12点前填写，未填写按前一日结算)。  
   * **DES/ARC 手动分配与销毁配置**。  
   * **私募额度修改**: 手动修改账号的DES/USDT私募额度。  
3. **用户管理**:  
   * 查看用户层级关系 (上级)。  
   * 查看/修改用户等级 (只能升级，不能降级)。  
   * 设置“社区办公室”地址。  
4. **公告中心**: 配置前端滚屏或弹窗公告。

## **8\. 前端用户界面需求 (Frontend)**

1. **首页/挖矿大厅**:  
   * 显示ARC/DES 实时价格 (K线或简报)。  
   * 挖矿入金入口。  
   * 个人算力、个人等级、累计收益展示。  
2. **资产/钱包**:  
   * ARC、DES、USDT 余额。  
   * **提现/领取页面**:  
     * 输入领取数量。  
     * **实时计算并展示**: 需扣除的DES数量。  
     * 文案提示: "ARC算力挖矿...3倍出局...禁止提取...提取需收取手续费..."。  
3. **社区/团队**:  
   * 显示直推人数、团队业绩。  
   * 社区办公室自行提币入口 (仅限办公室地址可见)。  
4. **交易 Swap**:  
   * ARC/USDT (买卖限制逻辑)。  
   * DES/USDT。

## **9\. 核心风险与注意点 (Tips for Devs)**

1. **价格预言机**: 提现手续费、入金销毁DES、入金销毁LP都需要实时获取ARC和DES对USDT的价格，需确保预言机或价格接口稳定，防止套利。  
2. **中心化依赖**: 每日产出依赖后台“主动计算”和“手动分配”，需开发自动化脚本辅助，否则人工操作风险极大。  
3. **精度处理**: 涉及多层级分润（10层、级差），需注意数值精度丢失问题。  
4. **Office寻址算法**: 社区办公室的递归查找（找最近的上级）需要高效的算法，避免由于层级过深导致Gas费过高或超时（建议后端计算，前端展示）。
